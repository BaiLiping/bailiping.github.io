## June
    1. 基于RadarScenes数据集
    - a). 用PointNet++实现Radar Detection Points的Semantic Segmentation以及后续的DBSCAN聚类与基于SVM/Random Forest的分类。
    - b). 基于PointNet++,修改其结构令其直接做end to end instance segmentation（而不是先semantic segmentation之后再DBSCAN聚类再分类），然后对比俩种方式的效果。 通过这部分熟悉怎么直接处理points提取feature的deep learning方案。
    2. 基于RADDet数据集，用YOLOv4做底板（这个代码我之后会给你）修改出来的object detector和直接使用RADDet论文中的object detection代码（这个网络比YOLOv4简单很多）这俩种方式（进行比较），实验Polar Coordinate和Cartesian Coordinate俩种不同format的range-azimuth heatmap（或者range-Doppler-azimuth cube）在同一种Obejct detector下的区别。通过这部分熟悉怎么用Range-Doppler-azimuth cube类的数据通过通用的object detection网络设计object detector并进行object detection。
    3. Multi-Target Tracking Filter部分的代码完成：
    - a). 完成JPDA的Python版本的仿真，并加入Track Management。
    - b). 完成GM-PHD的Python版本的仿真, 把除了配置GM-PHD的模块之外的function从GM_PHD_point_target.py文件取中，单独放入一个ultility文件中。并加入Gating。
    - c). 完成PMBM filter的Python版本的仿真。 通过这部分熟悉常用的random vector based和RFS based multi-target tracking方案。如果有余力，我们还可以在这些代码基础上完成CPHD, TCPHD, N-type PHD, TPMBM的方案（这些方案中除了N-type PHD比较复杂一些，其他的几种基本就是在原版上加一个非耦合的小模块，理解原理后不难实现）。
    4. 使用某种radar based object detector，加入uncertainty modelling的部分从而除了bounding box之外还估计bounding box的参数的variance（这种在object dwtector中建模bounding box的variance信息的方案有挺多的，比如可以在训练object detector时候给loss function加估计variance的各种idea。）。这种带uncertainty（variance）的object detector产生的bounding box作为measurement时可以同时提供一个更精确的measurement variance。那这样的话，我们的JPDA或者PHD这类filter里面的covariance matrix of measurement noise就可以不再需要是一个hardcoded的fixed value了，而变为measurement dependent的。所以我们再选择一种tracker(JPDA, PHD, PMBM等)，把带有measurement variance的bounding box作为measurement输入tracker，从而提供measurement dependent的更准确的covariance matrix of measurement noise。 那常见的加入uncertainty(variance)的方案包括： a). 对训练好的模型加随机dropout得到多个结果，统计每个像素的均值和方差得到variance。见论文:2020.Brain Tumor Segmentation using 3D-CNNs with Uncertainty Estimation.但这种方案可能只适合对每一个Pixel加入variance信息而不太适合bounding box regression问题。什么原因？因为bounding box的估计如果用随机dropout可能产生的bounding box都无法一一对应上，所以不能统计。 b). 对输入的训练数据进行采样得到多个patch，多次用不同patch训练数据训练得到多个模型，于是得到多个预测结果，统计每个像素的均值和方差得到variance.同样，这种方案可能只适合对每一个Pixel加入variance信息而不太适合bounding box regression问题。因为bounding box的估计如果用随机dropout可能产生的bounding box都无法一一对应上，所以不能统计。 c). 将ground truth值都当做dirac分布，然后把object detection网络输出部分输出两个变量，一个是每个像素的均值，另一个是每个像素的方差。然后通过minimize KL散度的方式得到一个MSE loss的变形，用这个loss来训练网络从而可以预测每个像素的方差。见论文：2019.Bounding Box Regression with Uncertainty for Accurate Object Detection.该KL loss的代码如下： elif cfg.boxloss == 'KL': l1_loss = respond_bbox * bbox_loss_scale * ( torch.exp(-pred_vari) * smooth_loss(target=label_coor, input=pred_coor) + 0.5 * pred_vari) * cfg.l1scale bbox_loss = l1_loss d). 和估计出的每个像素的均值方差表示的高斯分布很接近时候，ground truth的像素值代入用该均值和方差表示的高斯pdf函数应该Maximize，所以求sum(-log(Gaussian_pdf(i))),i表示每个像素。用这种方案得到每个像素的方差。见论文：Gaussian YOLOv3: An Accurate and Fast Object Detector Using Localization Uncertainty for Autonomous Driving. 除此之外，其他的tracker中hardcode的其他参数，比如covariance matrix for process noise，Ps，
    等等都可以考虑用一些别的思路去动态的根据场景和sensor的实际情况动态的表征，而不是直接hardcode一个fixed value。
